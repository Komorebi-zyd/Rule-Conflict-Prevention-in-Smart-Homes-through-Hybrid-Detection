\begin{abstract}

Smart home platforms rely on automation rules, yet interactions among rules may lead to unintended and unsafe behaviors. We propose a hybrid framework that combines offline static analysis with runtime dynamic monitoring for rule conflict detection. In the static phase, rules are modeled with a Trigger-Condition-Action-Environment (TCAE) representation, formal verification constructs a Rule Interaction Dependency Graph (RIDG), and the Entity Safety Configuration guides extraction of a Rule Conflict Dependency Graph (RCDG) that retains candidate conflict edges. At runtime, assertion verification checks whether a candidate conflict edge is activated under the current system state. Implemented on Home Assistant and evaluated in a multi-zone testbed with 17 automation rules, the framework detects all manually confirmed conflict cases under the configured Entity Safety Configuration, including cases involving physical channels and zone-dependent effects, while reducing false positives compared with IoTMediator and keeping runtime verification overhead at the millisecond level.

\keywords{Smart home · rule conflict · hybrid detection · runtime verification}

\end{abstract}
