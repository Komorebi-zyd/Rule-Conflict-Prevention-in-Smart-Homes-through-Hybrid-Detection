\section{Conclusion}

This paper presents a hybrid framework for rule conflict detection in smart home systems by combining offline static analysis with runtime dynamic monitoring. The framework extends automation rule modeling with physical-channel information, uses formal verification to construct the Rule Interaction Dependency Graph (RIDG), and applies the Entity Safety Configuration to extract the Rule Conflict Dependency Graph (RCDG), which retains the interaction edges corresponding to candidate rule conflicts. At runtime, assertion verification is used to check whether a candidate conflict edge is activated under the current system state.

Evaluation on a Home Assistant prototype shows that the framework can detect the manually confirmed conflict cases in the testbed under the configured Entity Safety Configuration, including cases involving physical channels and zone-dependent effects, while avoiding several non-conflict cases reported by IoTMediator. The measured execution time further indicates that static analysis can be performed offline and that the runtime overhead of assertion verification remains limited in our prototype.
